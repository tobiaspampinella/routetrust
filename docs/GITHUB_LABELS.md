# GitHub Labels

This repository uses a minimal, explicit label set defined in [`.github/labels.yml`](../.github/labels.yml).

## Core Labels

- `bug`
- `enhancement`
- `documentation`
- `security`
- `good first issue`
- `help wanted`
- `blocked`
- `needs review`
- `beta`

## Priority Labels

- `priority: critical`
- `priority: high`
- `priority: medium`
- `priority: low`

## Agent Labels

- `agent: codex`
- `agent: antigravity`
- `agent: opencode`
- `agent: qa`
- `agent: ux`

## Area Labels

- `area: cms`
- `area: demo`
- `area: maps`
- `area: telegram`
- `area: bug-assistant`
- `area: security`
- `area: github`
- `area: qa`
- `area: runtime`

## Usage Rules

- Every issue should have one area label.
- Every issue should have one priority label.
- Security issues should avoid public disclosure when the content is sensitive.
- Agent labels are optional and intended for ownership or handoff visibility.
