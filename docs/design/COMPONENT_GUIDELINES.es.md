# Component Guidelines

## Button

- las acciones primarias usan azul operacional
- las acciones destructivas usan rojo de incidencia
- los labels son cortos y verb-first

## MetricCard

- una metrica
- una linea de soporte
- un estado visible
- sin mezclar colores semanticos

## StatusBadge

- no inventar sinonimos
- mapear exactamente al modelo de estados

## EmptyState

- explicar que falta
- mostrar la siguiente accion util
- no culpar al usuario

## LoadingState

- nunca mostrar un panel vacio cuando se espera data

## ErrorState

- explicar el fallo de forma simple
- mostrar el impacto operacional

## LanguageToggle

- mantener ubicacion consistente de nivel superior
- no esconder el cambio de idioma dentro de settings en paginas publicas
