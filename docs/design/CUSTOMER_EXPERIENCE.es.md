# Checklist de Customer Experience

## Test de Cinco Segundos

El customer debe entender:

- donde esta el envio
- estado actual
- ETA
- si hay retraso
- si hay incidencia
- que sigue

## Checklist

- el header de tracking se lee de inmediato
- el ETA es visible above the fold en mobile
- el lenguaje de retraso es simple y no tecnico
- el mensaje de incidencia es claro y no alarmista
- el mensaje de entrega completada cierra bien el flujo
- el fallback del mapa sigue siendo util si el mapa live no esta disponible
- no hay clutter compitiendo con el estado del envio
