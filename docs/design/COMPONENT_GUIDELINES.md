# Component Guidelines

## Button

- primary actions use operational blue
- destructive actions use incident red
- labels are verb-first and short

## MetricCard

- one metric
- one support line
- one visible state
- no mixed semantic colors

## StatusBadge

- do not invent synonyms
- map to the state model exactly

## EmptyState

- explain what is missing
- show next useful action
- avoid blaming the user

## LoadingState

- never show a blank panel when data is expected

## ErrorState

- explain the failure plainly
- show operational impact

## LanguageToggle

- keep top-level placement consistent
- do not bury language switching inside settings for public pages
