# Design System

## Proposito

RouteTrust necesita un lenguaje visual operacional, no un tema generico de startup.

## Principios Base

- rapido de escanear
- baja carga cognitiva
- explicitamente operacional
- confiable bajo estres
- listo para bilingue

## Paleta

- graphite: `#12161D`
- graphite profundo: `#0B0F14`
- azul operacional: `#1D6BFF`
- verde exito: `#169C5A`
- amber advertencia: `#C98912`
- rojo incidencia: `#C53B2C`
- gris superficie: `#E6EBF2`

## Tipografia

- display: Geist
- body: Inter
- headings dashboard: jerarquia fuerte y compacta
- texto tracking customer: labels mas grandes y frases mas simples

## Componentes Base

- Button
- Card
- MetricCard
- StatusBadge
- AlertBanner
- Timeline
- RouteStatus
- DriverStatus
- IncidentCard
- ApprovalPanel
- EmptyState
- LoadingState
- ErrorState
- HelpTooltip
- LanguageToggle

## Modelo de Estados

- success
- warning
- incident
- delayed
- in progress
- completed
- blocked
- not configured
- demo only

## Gaps Actuales

- sesgo violeta visible en la UI actual
- copy mezclado entre idiomas
- sigue apareciendo mojibake en algunas superficies
- el set de rutas publicas de marketing esta incompleto
- el comportamiento de componentes todavia no esta estandarizado entre customer y admin
