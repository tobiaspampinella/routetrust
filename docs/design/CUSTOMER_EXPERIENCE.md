# Customer Experience Checklist

## Five-Second Test

The customer must understand:

- where the shipment is
- current status
- ETA
- whether there is a delay
- whether there is an incident
- what happens next

## Checklist

- tracking header is immediately readable
- ETA is visible above the fold on mobile
- delay language is plain and non-technical
- incident messaging is clear and non-alarming
- delivery-complete messaging closes the flow cleanly
- map fallback remains useful if live mapping is unavailable
- no clutter competes with shipment state
