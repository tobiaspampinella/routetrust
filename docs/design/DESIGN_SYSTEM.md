# Design System

## Purpose

RouteTrust needs an operational design language, not a generic startup theme.

## Core Principles

- fast to scan
- low cognitive load
- operationally explicit
- trustworthy under stress
- bilingual-ready

## Palette

- graphite: `#12161D`
- graphite deep: `#0B0F14`
- operational blue: `#1D6BFF`
- success green: `#169C5A`
- warning amber: `#C98912`
- incident red: `#C53B2C`
- surface gray: `#E6EBF2`

## Typography

- display: Geist
- body: Inter
- dashboard headings: strong, compressed hierarchy
- customer tracking text: larger labels, simpler sentence structures

## Base Components

- Button
- Card
- MetricCard
- StatusBadge
- AlertBanner
- Timeline
- RouteStatus
- DriverStatus
- IncidentCard
- ApprovalPanel
- EmptyState
- LoadingState
- ErrorState
- HelpTooltip
- LanguageToggle

## State Model

- success
- warning
- incident
- delayed
- in progress
- completed
- blocked
- not configured
- demo only

## Current Gaps

- visible purple bias on current UI
- mixed language copy
- mojibake still appears in some surfaces
- public marketing route set is incomplete
- component behavior is not yet standardized across customer and admin views
