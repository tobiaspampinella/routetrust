# GitHub Repository Presentation

## Public Repository Goals

- clear bilingual entry point
- honest status
- strong product framing
- discoverable docs
- no sensitive operational details

## README Requirements

- top bilingual switch
- short product definition
- current status
- product surfaces
- quickstart
- validation commands
- design-system and roadmap links

## Repository Hygiene

- remove unnecessary personal contact details from public docs
- keep security disclosure redacted
- avoid fake enterprise claims
- keep screenshots optional unless they are current

## Next Repository Tasks

- add badges only when they represent real checks
- add screenshots after browser QA capture exists
- keep issue and PR templates aligned with beta reality
