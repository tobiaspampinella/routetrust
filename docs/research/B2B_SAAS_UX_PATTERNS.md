# B2B SaaS UX Patterns

Generated: 2026-06-01

## Pattern Rules For RouteTrust

## Navigation

B2B SaaS navigation should be predictable:

- Dashboard
- Routes
- Drivers
- Tracking
- Incidents
- Approvals
- Reports
- Settings
- Admin/CMS
- Project status only for internal beta

## Onboarding

Required onboarding path:

1. Create tenant.
2. Configure operation region/language/SLA.
3. Invite users.
4. Import drivers.
5. Import or create routes.
6. Configure tracking page.
7. Run demo seed.
8. Approve go-live checklist.

## Admin CMS

CMS must separate:

- product content
- tenant settings
- operational rules
- approval policies
- notification settings
- audit logs

## Enterprise UX Requirements

- Role-based navigation.
- Clear permission errors.
- Auditability.
- Bulk actions.
- Filters and search.
- Export.
- Empty/loading/error states.
- Accessible status messages.
- Localization strategy.
- Demo mode labels.

## RouteTrust Gap

Current UI has many panels, but not enough workflow completion. The next UX work must prioritize workflows over more dashboard decoration.
