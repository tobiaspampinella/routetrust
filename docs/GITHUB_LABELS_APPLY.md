# GitHub Labels Apply

`gh` was not available on the publication machine, so labels were not applied automatically.

## Source Of Truth

- Local file: `.github/labels.yml`

## Manual Application

1. Open the GitHub repository.
2. Go to `Issues` -> `Labels`.
3. Create or update labels to match `.github/labels.yml`.

Required labels:

- `bug`
- `enhancement`
- `documentation`
- `security`
- `good first issue`
- `help wanted`
- `blocked`
- `needs review`
- `beta`
- `priority: critical`
- `priority: high`
- `priority: medium`
- `priority: low`
- `agent: codex`
- `agent: antigravity`
- `agent: opencode`
- `agent: qa`
- `agent: ux`
- `area: cms`
- `area: demo`
- `area: maps`
- `area: telegram`
- `area: bug-assistant`
- `area: security`
- `area: github`
- `area: qa`
- `area: runtime`

## Verification

After labels are created, compare the GitHub label list against `.github/labels.yml` and update this document if any deliberate deviations are introduced.
